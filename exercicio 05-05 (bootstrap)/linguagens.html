Linguagem de programação é igual como outras línguas que conhecemos?

Linguagem de programação é um conjunto de regras e símbolos que permitem aos programadores escrever instruções que os computadores podem entender e executar. Essas linguagens são usadas para criar software, aplicativos, jogos e sistemas operacionais. Elas fornecem uma maneira estruturada de comunicar-se com o computador, permitindo que os desenvolvedores especifiquem o que o computador deve fazer em termos de lógica, controle de fluxo e manipulação de dados. Então de certa forma sim, porém mais técnicas, pois são desenhadas para fazerem a ponte entre o humano e a máquina.

Existem muitas linguagens de programação, cada uma com suas próprias características e propósitos. Algumas são mais adequadas para desenvolvimento web, outras para aplicativos móveis, jogos ou sistemas embarcados. Exemplos populares incluem Python, JavaScript, Java, C++, Ruby e Swift. Cada linguagem tem sua própria sintaxe e semântica, o que significa que as regras de escrita e interpretação variam entre elas.

Surgimento das linguagens de programação

Uma das primeiras figuras que se destacam na história das linguagens de programação é Ada Lovelace, que é frequentemente considerada a primeira programadora do mundo. No século XIX, ela trabalhou com Charles Babbage na criação da máquina analítica, um precursor dos computadores modernos. Lovelace escreveu algoritmos para essa máquina, antecipando o conceito de programação e por isso recebeu uma homenagem com o nome de uma das linguagens de programação mais conhecidas, a Ada.
No entanto, as linguagens de programação modernas começaram a surgir na década de 1950, com o desenvolvimento de linguagens como Fortran (Formula Translation) e COBOL (Common Business Oriented Language). Essas linguagens foram projetadas para facilitar a programação e torná-la mais acessível a um público mais amplo. Desde então, muitas outras linguagens foram criadas, cada uma com suas próprias características e propósitos específicos. Depois houve o surgimento da linguagem Lisp, voltada para pesquisa em inteligência artificial e processamento simbólico, marcando mais um avanço importante na evolução das linguagens de programação.
A orientação a objetos é outro marco importante na história das linguagens de programação. A linguagem Simula 67 introduz o conceito de classes. A linguagem Smalltalk expande o conceito de classes e se torna a primeira linguagem de programação que oferecia suporte completo à programação orientada a objetos. A linguagem C++ (originalmente conhecida como C com classes) populariza a orientação a objetos.

Como funcionam?

Uma linguagem de programação pode ser convertida, ou traduzida, em código de máquina por compilação ou interpretada por um processo denominado interpretação. Em ambas ocorre a tradução do código fonte para código de máquina.
A compilação é o processo de traduzir todo o código fonte de um programa em um arquivo executável antes de ser executado. Isso significa que o código é convertido em código de máquina, que é diretamente compreendido pelo processador do computador. Linguagens como C, C++ e Rust são exemplos de linguagens compiladas.
A interpretação, por outro lado, envolve a tradução do código fonte em tempo real, linha por linha, enquanto o programa está sendo executado. Isso significa que o código é interpretado e executado simultaneamente. Linguagens como Python, JavaScript e Ruby são exemplos de linguagens interpretadas.

Classificação, quanto ao grau de abstração

Linguagens podem ser classificadas quanto ao grau de abstração, que é a distância entre a linguagem e o hardware do computador. As linguagens de baixo nível são mais próximas do hardware e exigem um conhecimento mais profundo da arquitetura do computador, tais como: Binário e Assembly. Enquanto as linguagens de alto nível são mais abstratas e fáceis de usar, permitindo que os programadores se concentrem mais na lógica do programa do que nos detalhes técnicos, como Pascal e SQL. No meio termo estão as linguagens de médio nível, que oferecem um equilíbrio entre abstração e controle sobre o hardware, como C e C++.
